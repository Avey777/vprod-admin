实例 > 组织/租户/团队 > 项目> 应用 > 客户端

一个核心用户可以加入多个团队


```
                    租户1 (Tenant : 组织/个人/团队)
                      |
                      |── 角色1 (管理员) → 访问应用X, 应用Y
                      |
                    项目1 (Project)
                      |
                      |
        -----------------------------------
        |                                 |
      应用1 (Application 1)                应用2 (Application 2)
        |                                 |
        |                                 |
-------------------               -------------------
|       |        |               |        |        |
Web    App    小程序            Web     API      ...
(终端入口)  (终端入口) (终端入口)    (终端入口)(终端入口)
```


1. 用户可以属于多个团队（多对多）
2. 一个团队有可以订阅多个应用（多对多）
3. 一个应用可以被一个团队多次订阅（根据应用设置）
4. 一个角色可以关联多个同团队的应用以及权限
