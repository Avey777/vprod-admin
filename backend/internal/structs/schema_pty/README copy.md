实例 > 组织/租户/团队 > 项目> 应用 > 客户端

一个核心用户可以加入多个团队



```
                    团队1 (Team)
                      |
                      |── 角色1 (管理员) → 访问应用X, 应用Y:门户Z:客户端:V
                      |
                    应用1 (Application)
                      |
                      |
        -----------------------------------
        |                                 |
      门户1 (Portal 1)                门户2 (Portal 2)
        |                                 |
        |                                 |
-------------------               -------------------
|       |        |               |        |        |
Web    App    小程序            Web     API      ...
(终端入口)  (终端入口) (终端入口)    (终端入口)(终端入口)
```


1. 用户可以属于多个团队（多对多）
2. 一个团队有多个应用（多对多）
3. 一个应用有多个门户（一对多）
4. 一个角色可以授权多个同团队的门户（注意：角色与门户的关系，且要求门户属于同一个团队）

级联删除

删除应用 → 删除团队应用关联、应用门户
删除角色 → 删除角色门户授权、用户角色关联
